UI/UX comparison draft

Embedded SW Tester 워크벤치 개선 목업/비교안

현재 제품 방향은 대시보드가 아니라 장비 제어와 테스트 디버깅을 동시에 수행하는 IDE입니다. 따라서 개선의 핵심은 화면을 화려하게 바꾸는 것이 아니라, 실행 상태와 편집 상태를 분리해서 읽히게 하고, YAML/GUI/Trace/Variables가 같은 맥락을 공유한다는 느낌을 만드는 것입니다.

추천 방향: A 기반 + B/C 일부 흡수

결론

  • A안으로 전체 워크벤치 골격과 상태 표시를 정리합니다.
  • B안의 Trace Inspector를 실행 중 분석 UX에 반영합니다.
  • C안의 블록 선택/Properties 흐름을 GUI 편집 UX에 반영합니다.

현재 UX에서 우선 풀어야 할 문제

영역 현재 사용자가 느끼는 문제 개선 원칙
상단 메뉴/툴바 실행, 일시정지, resume, breakpoint, 저장, 설정이 한 줄에 섞여 있어 현재 작업 맥락이 약합니다. 명령 그룹을 나누고, 현재 YAML/실행 대상/저장 상태를 별도 Context Strip으로 고정합니다.
YAML Editor 현재 실행 줄은 보이지만 GUI 선택, Trace 선택, Breakpoint와의 관계가 한눈에 정리되기 어렵습니다. 현재 실행 영역은 배경, 사용자가 선택한 블록은 테두리, breakpoint 가능 라인은 gutter UI로 역할을 분리합니다.
GUI Testcase Editor 명령 추가/이동/선택이 가능해졌지만, 많은 블록을 볼 때 중요한 명령과 보조 명령의 시각적 우선순위가 같습니다. Delay 같은 보조 명령은 row처럼 낮게 보이게 하고, for/if/call/assert/adapter 명령을 중심 블록으로 둡니다.
Properties 명령별 필수값, optional값, action별 조건부 필드가 많은데 한 번에 보이면 부담스럽습니다. 필수값 우선, action별 필드 노출, optional은 접기/설정 기본값을 사용합니다.
Execution Trace 실행 중 빠르게 지나가는 이벤트를 놓치기 쉽고, 각 컬럼의 의미를 처음 보는 사용자가 바로 알기 어렵습니다. Follow latest와 Pin detail을 분리하고, Trace row 선택 시 YAML/GUI/Variables/Console을 연결합니다.

3가지 화면 구성 비교

A안 추천

Stabilized IDE Shell

현재 구조를 유지하면서 IDE의 정보 계층을 정리하는 안입니다.

학습 비용
구현 난이도
확장성
  • 현재 WPF 구조와 가장 잘 맞습니다.
  • 도킹 UI로 확장하기 전의 안정적인 중간 단계입니다.
  • 작업 상태가 상단/하단으로 명확히 분리됩니다.
B안

Debug-First Trace Console

실행 결과 분석과 장비 연동 디버깅을 중심에 두는 안입니다.

학습 비용
구현 난이도
확장성
  • 장비 smoke/debug 시 가장 강합니다.
  • Trace, Console, Variables가 중심 화면이 됩니다.
  • 저작 경험은 상대적으로 뒤로 밀릴 수 있습니다.
C안

Block-First Authoring Studio

YAML을 잘 모르는 사용자도 GUI 블록으로 테스트를 작성하기 쉽게 하는 안입니다.

학습 비용
구현 난이도
확장성
  • 신규 사용자에게 가장 쉽습니다.
  • Properties와 자동완성 품질이 중요합니다.
  • YAML 직접 편집 파워 유저에게는 답답할 수 있습니다.

추천 구현 순서

1

Workbench Shell 정리

상단 메뉴/툴바/Context Strip/상태바를 정리하고 현재 YAML, 실행 대상, 저장 상태, 현재 line을 항상 보이게 합니다.

2

편집 선택 모델 정리

YAML 선택, GUI 블록 선택, Trace 선택 중 하나만 active selection이 되게 하고 나머지는 linked highlight로 낮춥니다.

3

Trace Inspector 강화

Follow latest, Pin detail, row tooltip, resolved input/output/error 상세 표시를 하나의 inspector UX로 묶습니다.

4

Properties 사용성 강화

필수값 우선 표시, action별 조건부 필드, optional 접기, 자동완성 후보를 명령 카탈로그와 연결합니다.
